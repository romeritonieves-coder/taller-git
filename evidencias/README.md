# Evidencias del Taller

Cada estudiante debe tomar capturas de pantalla de las siguientes evidencias:

## Evidencias Técnicas

1. Rama de trabajo creada
2. Archivo asignado modificado
3. Cambios publicados en GitHub
4. Actualización desde el repositorio remoto
5. Historial antes y después de reorganizar la rama
6. Evidencia de conflicto, si ocurrió
7. Evidencia de solución del conflicto
8. Historial antes y después de limpiar commits
9. Evidencia de incorporación de un cambio específico desde otra rama
10. Pull Request final

## Evidencias Conceptuales

1. Explicación del flujo realizado
2. Justificación de las decisiones tomadas
3. Descripción de problemas encontrados
4. Reflexión individual
5. Conclusiones del equipo

## Comandos Útiles para Capturas

```bash
git status
git branch
git branch -a
git log --oneline --graph --all
git diff
git log --oneline -10
```

## Formato de Archivos

Nombrar las capturas con el siguiente formato:
- evidencia-XX-nombre-descriptivo.png

Ejemplo: evidencia-01-rama-creada.png
